考え方
01

自分の Workspace で動く。

ドキュメントもデータも、あなたのインフラの中。誰かのクラウドに預けるのではなく、自分の space の中で生まれて、自分の space の中に残る。

02

エージェントが、書く。

すべてのアプリが MCP を話す。AI エージェントがあなたの文書・スライド・シートを直接読み書きできる。「議事録をまとめて」「この表をグラフ用に整えて」が、そのまま動く。

03

サブスクではなく、Capsule。

席数ライセンスも、月額もない。OpenTofu Capsule として自分のインフラに install するだけ。使わなくなったアプリは、いつでも uninstall できる。

Agent-native

あなたの代わりに、
道具を動かす。

Takos Office の各アプリは MCP サーバーを公開している。Claude のような MCP 対応エージェントが、あなたのドキュメントを直接編集できる。

「先週のメモから提案書のスライドを作って」と頼めば、エージェントが slide を開いて組み立てる。コピペも、アプリの切り替えもいらない。

docs · slide · excel — すべて /mcp endpoint を持つ

agent → takos-slide /mcp
# エージェントが MCP ツールを呼ぶ tool: slide.create_deck args: title: "Q3 提案書" from: "docs://notes/last-week" slides: 8 # → あなたの Workspace の中で # → スライドが組み上がる result: "deck_3f9a" created ✓
